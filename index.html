Project Title

BloodLink 🩸

A comprehensive digital solution for managing blood donors, tracking inventory levels, and generating analytical reports for blood banks.

🩸 BloodLink is a Java-based desktop application built with Java Swing and MySQL. It implements role-based access control, real-time data synchronization, and a modern glassmorphism UI designed for healthcare professionals.
Platform: Java Desktop App
Database: MySQL (XAMPP)
Framework: Java Swing
Group: Group 1
Security: PreparedStatement + RBAC
03 / 13
Overview

Key Features

BloodLink streamlines blood bank operations through seven integrated modules. Each feature is designed with user experience and data integrity in mind.
Dashboard Overview
Real-time statistics: total donors, blood units, critical alerts, and recent donations with live clock.
Role-Based Access
Two-tier system: Admin (full access) and Staff (restricted) with session management via UserSession.java.
Donor Registration
Collect donor details with validation. Auto-updates inventory upon successful registration.
Donor Management
Full CRUD operations with search/filter capabilities using SQL LIKE queries.
Inventory Tracking
Real-time stock monitoring with color-coded CRITICAL alerts when units drop below threshold.
Reports & Analytics
Monthly trends and distribution reports for easy tracking and analysis of blood bank data.
System Settings
Admin-exclusive user management: add/delete accounts, password changes, system info display.

Database Integration

SQL Statements

LOGIN — Verify Credentials
SELECT * FROM users
WHERE username = ? AND password = ?
-- ? = PreparedStatement placeholder (SQL Injection Protected)
INSERT — New Donor
INSERT INTO donors (name, age, blood_group, phone, address, donation_date)
VALUES (?, ?, ?, ?, ?, ?)
SEARCH — Filter Donors
SELECT * FROM donors
WHERE name LIKE ? OR blood_group LIKE ?
ORDER BY id DESC
-- % wildcard matches any characters (e.g., '%O+%' finds all O+ donors)
UPDATE — Edit Donor
UPDATE donors
SET name=?, age=?, blood_group=?, phone=?, address=?
WHERE id = ?
-- WHERE id = ? ensures only selected donor is updated
DELETE — Remove Record
DELETE FROM donors
WHERE id = ?
-- Permanently removes the record from the database
SELECT — Display All Donors
SELECT id, name, age, blood_group, phone, address, donation_date
FROM donors
ORDER BY id DESC
-- ORDER BY id DESC = newest records appear first
FILTER — Critical Stock
SELECT blood_group, available_units
FROM blood_inventory
WHERE available_units < 5
ORDER BY blood_group ASC
-- Filters blood types with CRITICAL stock (< 5 units)
REPORTS — Monthly Trends
SELECT MONTHNAME(donation_date) as month,
       YEAR(donation_date) as year,
       COUNT(*) as total_donations
FROM donors
GROUP BY YEAR(donation_date), MONTH(donation_date)
ORDER BY year DESC, month DESC
AUTO-UPDATE — Inventory Sync
UPDATE blood_inventory
SET available_units = available_units + 1
WHERE blood_group = ?
-- Automatically adds 1 unit when new donor is registered
